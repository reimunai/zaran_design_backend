## 1. 接口设计规范

### 1.1 通用约定

| 项目 | 规范 |
|------|------|
| **协议** | HTTPS |
| **数据格式** | JSON (UTF-8) |
| **认证方式** | `Authorization: Bearer <accessToken>` |
| **时间格式** | ISO 8601 (`2026-05-31T14:30:00+08:00`) |
| **分页参数** | `page` (默认1), `size` (默认10, 最大50) |
| **排序参数** | `sort` 字段，如 `createdAt,desc` |

### 1.2 HTTP 方法语义

| 方法 | 用途 |
|------|------|
| `GET` | 查询资源 |
| `POST` | 创建资源 |
| `PUT` | 全量更新资源 |
| `PATCH` | 部分更新资源 |
| `DELETE` | 删除资源 |

### 1.3 路径设计原则

- 资源名词使用复数形式，如 `/users`, `/sketches`
- 层级关系使用路径嵌套，如 `/sketches/{id}/versions`
- 动作型操作使用动词，如 `/auth/login`, `/tasks/{id}/cancel`

**Base URL**: `/auth`

### 2.1 接口列表

| 序号 | 方法 | 路径 | 说明 | 认证 |

## 4. 草图绘制模块 (Sketch)

**Base URL**: `/sketches`

### 4.1 接口列表

| 序号 | 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|------|
| 1 | `POST` | `/sketches` | 创建新草图 | 是 |
| 2 | `GET` | `/sketches` | 查询我的草图列表 | 是 |
| 3 | `GET` | `/sketches/{sketchId}` | 获取草图详情 | 是 |
| 4 | `PUT` | `/sketches/{sketchId}` | 保存草图 | 是 |
| 5 | `PATCH` | `/sketches/{sketchId}/visibility` | 设置草图公开/私密 | 是 |
| 6 | `DELETE` | `/sketches/{sketchId}` | 删除草图（软删除） | 是 |
| 7 | `POST` | `/sketches/{sketchId}/recover` | 恢复已删除草图 | 是 |
| 8 | `POST` | `/sketches/{sketchId}/versions` | 创建版本快照 | 是 |
| 9 | `GET` | `/sketches/{sketchId}/versions` | 获取版本历史 | 是 |
| 10 | `GET` | `/sketches/{sketchId}/versions/{versionId}` | 获取指定版本详情 | 是 |
| 11 | `POST` | `/sketches/{sketchId}/versions/{versionId}/restore` | 回退到指定版本 | 是 |
| 12 | `POST` | `/sketches/{sketchId}/import` | 导入本地图片 | 是 |
| 13 | `GET` | `/sketches/categories` | 获取草图分类树 | 否 |

> **说明**：草图数据对应数据库 `sketches`、`sketch_versions`、`sketch_categories` 三张表。图层矢量数据统一存储在 `layers_json` 字段中，缩略图（`thumbnail_path`）由后端在保存时根据 `imageBase64` 自动生成并上传至 MinIO；删除采用软删除（写入 `deleted_at`），默认列表不返回已删除草图。

### 4.2 详细接口说明

#### 4.2.1 创建新草图

创建一张空白画布草图，仅写入元数据，`layers_json` 初始为空，可后续通过「保存草图」接口填充图层数据。

```http
POST /api/sketches
Authorization: Bearer <accessToken>
Content-Type: application/json
```

**请求参数:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 否 | 草图名称，缺省为「未命名草图」 |
| `width` | int | 是 | 画布宽度（像素），范围 512-4096 |
| `height` | int | 是 | 画布高度（像素），范围 512-4096 |
| `categoryId` | int | 否 | 分类ID，对应 `sketch_categories.category_id` |

**请求示例:**

```json
{
  "name": "云纹草图v1",
  "width": 1024,
  "height": 1024,
  "categoryId": 1
}
```

**响应示例:**

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "sketchId": 1001,
    "name": "云纹草图v1",
    "width": 1024,
    "height": 1024,
    "categoryId": 1,
    "isPublic": false,
    "createdAt": "2026-05-31T14:30:00+08:00"
  }
}
```

**错误码:**

| 错误码 | 说明 |
|--------|------|
| `400` | `width`/`height` 超出 512-4096 范围 |
| `404` | `categoryId` 对应分类不存在 |

```http

Authorization: Bearer <accessToken>
```

#### 4.2.4 保存草图

全量更新草图的图层数据与基本信息。后端根据 `imageBase64` 自动生成并覆盖缩略图（`thumbnail_path`）。该接口仅更新当前草图主记录，不会自动产生版本快照；如需保留历史，请调用「创建版本快照」接口。

```http
PUT /api/sketches/{sketchId}
Authorization: Bearer <accessToken>
Content-Type: application/json
```

**请求参数:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 否 | 草图名称 |
| `layersJson` | object | 是 | 图层结构数据（矢量路径、栅格图层等） |
| `imageBase64` | string | 否 | 预览图Base64，用于生成缩略图；自动保存时可省略 |
| `categoryId` | int | 否 | 修改分类 |
| `isAutoSave` | boolean | 否 | 是否为前端定时自动保存，默认false |
| `tags` | string[] | 否 | 标签数组 |

**请求示例:**

```json
{
  "name": "云纹草图v3",
  "layersJson": {
    "version": "1.0",
    "layers": [
      {
        "id": "layer_1",
        "name": "背景",
        "type": "raster",
        "visible": true,
        "opacity": 1.0,
        "data": "base64_encoded_image_data..."
      },
      {
        "id": "layer_2",
        "name": "线条层",
        "type": "vector",
        "visible": true,
        "paths": [
          {"d": "M10,10 L100,100", "stroke": "#000000", "width": 2}
        ]
      }
    ],
    "canvas": {"width": 1024, "height": 1024}
  },
  "imageBase64": "data:image/png;base64,iVBORw0KGgo...",
  "isAutoSave": false,
  "tags": ["云纹", "草稿"]
}
```

**响应示例:**

```json
{
  "code": 200,
  "message": "保存成功",
  "data": {
    "sketchId": 1001,
    "name": "云纹草图v3",
    "thumbnailUrl": "https://minio.zaran.com/sketch/1001_thumb.png",
    "updatedAt": "2026-05-31T15:30:00+08:00"
  }
}
```

**错误码:**

| 错误码 | 说明 |
|--------|------|
| `4042` | 草图不存在或已删除 |
| `4031` | 非草图所有者，无法保存 |
| `5002` | 缩略图上传 MinIO 失败 |

## 12. 全局响应规范与错误码

### 12.1 响应结构

```json
// 成功响应（单条数据）
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": 1717075200000
}

// 成功响应（列表数据）
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ... ],
    "total": 156,
    "page": 1,
    "size": 10,
    "pages": 16
  },
  "timestamp": 1717075200000
}

// 成功响应（无数据）
{
  "code": 200,
  "message": "操作成功",
  "data": null,
  "timestamp": 1717075200000
}

// 错误响应
{
  "code": 4001,
  "message": "生成任务参数错误：k值必须是2、3或4",
  "data": null,
  "timestamp": 1717075200000
}
```

### 12.2 HTTP状态码

| 状态码 | 含义 | 场景 |
|--------|------|------|
| `200` | OK | 请求成功 |
| `201` | Created | 资源创建成功 |
| `400` | Bad Request | 请求参数错误 |
| `401` | Unauthorized | 未提供Token或Token无效 |
| `403` | Forbidden | 无权限访问该资源 |
| `404` | Not Found | 资源不存在 |
| `429` | Too Many Requests | 请求过于频繁，触发限流 |
| `500` | Internal Server Error | 服务器内部错误 |

### 12.3 业务错误码

| 错误码 | 含义 | 说明 |
|--------|------|------|
| `200` | 成功 | 操作成功 |
| `400` | 请求参数错误 | 通用参数校验失败 |
| `4001` | 生成参数错误 | k值、noiseLevel、patchMode不合法 |
| `4002` | 文件格式错误 | 上传文件非PNG/JPG/SVG |
| `4003` | 文件过大 | 上传文件超过20MB限制 |
| `401` | 未登录 | Token缺失或过期 |
| `4011` | Token过期 | AccessToken已过期，需刷新 |
| `4012` | RefreshToken过期 | 需重新登录 |
| `403` | 无权限 | 当前角色无法执行该操作 |
| `4031` | 非资源所有者 | 无法修改他人资源 |
| `404` | 资源不存在 | 查询的ID对应资源不存在 |
| `4041` | 用户不存在 | 用户ID无效 |
| `4042` | 草图不存在 | 草图ID无效或已删除 |
| `4043` | 任务不存在 | 生成任务ID无效 |
| `4044` | 作品不存在 | 图案ID无效或已删除 |
| `409` | 资源冲突 | 重复收藏、重复点赞等 |
| `429` | 请求过于频繁 | 触发接口限流 |
| `500` | 服务器内部错误 | 通用服务器错误 |
| `5001` | AI模型服务调用失败 | FastAPI服务异常 |
| `5002` | 文件上传失败 | MinIO/OSS上传异常 |
| `5003` | 生成任务队列已满 | 当前队列长度超过最大容量 |
| `5004` | 数据库操作失败 | SQL执行异常 |
| `5005` | Redis操作失败 | 缓存服务异常 |
| `5006` | 消息队列异常 | RabbitMQ发送失败 |
| `5007` | WebSocket连接失败 | 实时通信服务异常 |
